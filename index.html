Projects

Grains Detection

A machine learning project while doing a student job at a university research institute BIK. The objective is to use computer vision technology to detect whether the grains (animal feed) passing through the conveyor belt are sufficiently crushed and thus adjust the parameters of the crusher.

My main task is to retrain multiple neural network models (SSD, RCNN) with the grains images using Tensorflow's object detection API. Making the models capable of detecting and classifying four different grains (Triticale, Roggen, Weizen and Mais) accurately in the images. Wrapping the well-trained models and runtime environment in docker images and deploying them to a server for real-time detection.

Waste Material Classification

My other machine learning project at the institute BIK. Attempt to classify five waste materials (Cardboard, Shredded Paper, Plastic Foil, and Pamphlet) using machine learning methods.

Semi-supervised and unsupervised learning methods were used in this project to classify the waste material images. Test the semi-supervised learning method proposed by Philip Haeusser to classify the current dataset. Clustering the dataset using unsupervised methods such as autoencoder. Analyze clustering performance by dimension reduction and visualizing image embeddings using multiple algorithms (LDA, PCA, t-SNE).

Heart Rate Clock

Helping an art school student implement her digital artwork project, a clock that runs according to a person's heart rate. For every beat of the heart, the second hand of this clock will move. An adult's heart rate in a calm state is 60 beats per minute. If the user's heartbeat is at 120 beats per minute, the clock will go twice as fast as normal.

An apple watch monitors the heart rate, and the heart rate data is continuously sent to a server written in Nodejs via HTTP requests. In order to display the time in multiple places, the clock's interface is a web page written in Vuejs. This time needs to be displayed not only on this web page but also on a mechanical clock. Therefore, the MQTT protocol is used to communicate between the client and the server. This project showed me how difficult it is to develop iOS applications, especially when this application needs to collect user data. We only have a first-generation apple watch and an iPhone 5s, and it isn't easy to develop apps for these two products in 2020.
